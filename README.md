# DuAxMPNN-Chi

DuAxMPNN-Chi is a JAX/Equinox/Optax toolkit for training **TriAxMPNN** — a
chirality-extended tri-axis message-passing neural network on the
[PCQM4Mv2](https://ogb.stanford.edu/docs/lsc/pcqm4mv2/) molecular graph
benchmark. It predicts HOMO-LUMO gaps in eV and reports validation MAE.

What DuAxMPNN-Chi includes:

- `src.dataset.dataprocess`: raw SMILES plus SDF coordinates → HDF5 graph tensors with chiral columns.
- `src.dataset`: HDF5 loading, train-split coord filtering, and collapsed graph batching.
- `src.model`: the fixed `TriAxMPNN` C1 production architecture.
- `src.optim`: Adan optimizer, per-parameter LR/WD multipliers, and geometric LR scheduling.
- `src.train`: training loop with chiral auxiliary loss, validation, and checkpoint saving.

## Quick Start

```bash
# Install runtime dependencies.
python -m pip install -r requirements.txt

# dataprocess.py imports pandas; install it if your environment does not already provide it.
python -m pip install pandas

# Build processed graph tensors (requires SDF for chiral volume features).
python -m src.dataset.dataprocess --h-mode active

# Train TriAxMPNN and save the best validation checkpoint.
python -m src.train --model_save_path results/best_model.eqx

# Run unit tests.
python -m pytest
```

## Data Layout

By default, commands resolve paths from the current working directory and expect this layout:

```text
dataset/
  pcqm4m-v2/
    raw/
      data.csv.gz              # columns: smiles, homolumogap
      pcqm4m-v2-train.sdf      # 3D coordinates (required for chiral aux on train split)
    split_dict.h5              # train/valid/test split indices
    processed/
      data_processed.h5        # generated by dataprocess.py
```

Hydrogen handling is controlled by `--h-mode`:

- `active` → `processed/data_processed.h5` (default)
- `all` → `processed/data_processed_all.h5`
- `heavy` → `processed/data_processed_heavy.h5`

Use `--overwrite` when regenerating an existing processed file.

```bash
python -m src.dataset.dataprocess \
  --dataset-root dataset \
  --dataset-name pcqm4m-v2 \
  --h-mode active \
  --overwrite
```

## Training

```bash
python -m src.train \
  --batch_size 384 \
  --learning_rate 2e-3 \
  --weight_decay 1e-2 \
  --scheduler_period 8 \
  --model_save_path results/best_model.eqx \
  --seed 0
```

The CLI trains for `scheduler_period ** 2` epochs. With the default
`--scheduler_period 8`, that is 64 epochs. The best checkpoint by validation
MAE is serialized with `eqx.tree_serialise_leaves`.

Training arguments:

| Argument | Default | Description |
|---|---:|---|
| `--batch_size` | `384` | Real graphs per batch. A null graph is prepended internally for padding. |
| `--learning_rate` | `2e-3` | Peak learning rate. |
| `--weight_decay` | `1e-2` | Base weight decay before per-parameter multipliers. |
| `--scheduler_period` | `8` | Period length `k`; total epochs are `k ** 2`. |
| `--model_save_path` | `results/best_model.eqx` | Best-checkpoint output path. |
| `--seed` | `0` | RNG seed for model init, training shuffle, and dropout. |
| `--h-mode` | `active` | Chooses the processed HDF5 variant unless `--processed-h5` is set. |
| `--processed-h5` | `None` | Explicit processed HDF5 basename under `processed/`. |

Architecture and chirality are fixed in `src.model` (C1 production stack; no CLI toggles).

## Model

`get_model()` creates `TriAxMPNN(depth=5, width=256, num_head=16, dim_head=16)`.

Fixed C1 configuration:

- `max_hops=4`, `depth_mode=dense`
- RBF-linear edge electronic embedding, `elec_mode=edge_diff`, LoRA edge fuse
- Chiral message on 1-hop conv, 3D-gated chirality input, calibrated channel sampling
- Train-only signed-volume auxiliary loss (`chiral_aux_weight=0.1`)
- No additive CIP-sign node feature at the encoder

Key components:

- `FloatEmbedLayer` (RBF-linear) embeds edge electronegativity differences.
- `Chiral3DInputEmbed` gates continuous 3D signed tetrahedral volume during training.
- `ConvKernel` with multiplicative chiral message on the 1-hop convolution.
- `LayerMixerKernel` and `DepthMixerKernel` for bond-axis and depth-axis mixing.
- `chiral_head` distills 3D volume on the train split only.

Primary loss is gap MAE; training adds a masked volume MAE auxiliary term.

## Dataset API

```python
from src.dataset import PCQMDataset, PCQMDataloader

dataset = PCQMDataset(
    dataset_root="dataset/pcqm4m-v2",
    split="train",
    load_in_memory=True,
)

loader = PCQMDataloader(
    dataset,
    batch_size=384,
    shuffle=True,
    drop_last=True,
    seed=0,
)

for batch in loader:
    node_feat = batch["node_feat"]      # int32, shape (N, 10)
    node_embd = batch["node_embd"]      # float32, shape (N, 19)
    edge_idx = batch["edge_index"]      # int32, shape (2, E)
    labels = batch["labels"]            # float32, shape (B,)
    break
```

The train split applies coord filtering so only graphs with full 3D coordinates
are used (required for the chiral auxiliary loss). Valid and test splits are
not filtered.

## Batch Format

Collapsed batches concatenate multiple molecules into one padded graph block.
Graph index `0` is a zero-filled null graph; real graph indices start at `1`.

`node_embd` has 19 continuous columns:

```text
[RWPE(0:12), coord(12:15), en(15), gc(16), chiral_sign(17), chiral_vol(18)]
```

Core keys:

| Key | dtype | Shape | Notes |
|---|---|---|---|
| `node_feat` | `int32` | `(N, 10)` | Discrete atom features with offsets applied. |
| `node_embd` | `float32` | `(N, 19)` | RWPE, coords, EN/GC, CIP sign, signed tetra volume. |
| `node_ptr` | `int32` | `(B + 2,)` | Node boundaries for null plus real graphs. |
| `node_batch` | `int32` | `(N,)` | Graph index per node. |
| `edge_index` | `int32` | `(2, E)` | Direct-bond edge indices with graph offsets applied. |
| `edge_feat` | `int32` | `(E, 6)` | Direct-bond features with offsets applied. |
| `edge_2hop_*` | mixed | varies | 2-hop edge index, feature, ptr, and batch arrays. |
| `edge_3hop_*` | mixed | varies | 3-hop edge index, feature, ptr, and batch arrays. |
| `edge_4hop_*` | mixed | varies | 4-hop edge index, feature, ptr, and batch arrays. |
| `labels` | `float32` | `(B,)` | HOMO-LUMO gap in eV. |
| `molecule_ids` | `int64` | `(B,)` | Original dataset row indices. |
| `batch_n_graphs` | `int32` | scalar | Number of real graphs, excluding the null graph. |

`chiral_vol` is used only during training for the auxiliary loss; inference
uses CIP sign and gated 3D features according to the fixed calibrated schedule.
